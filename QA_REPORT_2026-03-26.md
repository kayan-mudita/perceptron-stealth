## BUGS FOUND & FIXED DURING TESTING

### Critical (Fixed)
1. **DB Schema Drift** — `User.stripePriceId`, `BrandKit` table, `LifecycleEvent` table, `Photo.photoAnalysis` column missing from database. All API routes were 500-ing. Fixed with `prisma db push` + direct SQL.

2. **Shotstack trim on wrong object** — `trim` was placed on the `clip` object but Shotstack requires it on the `asset` object. All stitch jobs failed with 400. Fixed in `video-stitcher.ts`.

3. **Shotstack "done" status not handled** — Shotstack sandbox returns `"done"` but code only checked for `"completed"`. `poll_stitch` looped forever. Fixed in `stitch-poll.ts` and `video-stitcher.ts`.

4. **CharacterSheet.updatedAt NULL constraint** — DB had `updatedAt` column without default; Prisma schema omits it. New character sheet creation crashed. Fixed with `ALTER TABLE ... ALTER COLUMN "updatedAt" SET DEFAULT NOW()`.

### High Priority (Fixed)
5. **Usage limit blocks dev testing** — `canGenerate: false` because Stripe not configured + 23 videos already generated. Fixed in `usage.ts`: dev mode now returns unlimited.

6. **GET /api/videos/[id] missing** — No GET handler on single video endpoint. Pipeline UI couldn't check individual video status. Added GET handler.

### Medium Priority (Not Fixed - Noted)
7. **POST /api/generate accepts empty script** — No server-side validation requiring script content. Empty script videos get created and enter pipeline.

8. **POST /api/brand-kit returns 405** — Only PATCH/GET supported; POST returns Method Not Allowed. Confusing DX.

9. **Posting-package LinkedIn caption uses raw script** — Generated LinkedIn copy contains the cut-level prompt text rather than actual finished content.

10. **All photos lack photoAnalysis** — 119 photos in DB, 0 have photoAnalysis populated. Analysis runs at upload time but isn't persisting.
