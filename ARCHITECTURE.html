UI Only

AI Video Generation

The core product. Users describe what they want in a chat interface. The AI "teammate" generates videos using their uploaded photos + voice samples. Two AI models: Kling 2.6 (hyper-realistic) and Seedance 2.0 (creative).

POST /api/generate Video model Kling 2.6 API

Industry templates pre-loaded: Real Estate (8 templates), Legal, Medical, Creator. Users can also paste Google Reviews and convert them into video testimonials.

→ Backend API exists but UI uses simulated setTimeout responses, not real API calls

Approval Workflow

Carousel-style review of pending videos. Each shows the full AI script, model used, duration, and content type. Three actions: Approve (green), Request Changes (yellow), Reject (red). Optional feedback field + schedule-after-approval.

PATCH /api/videos/[id] Video.status

Great UX pattern — but approval actions are button-only with no API calls. The status field exists on the Video model to support this.

→ Wire up: PATCH video status + create Schedule on approve

Data Architecture
User ─┬─ has one ──── BrandProfile  (name, tagline, tone, audience, colors, guidelines)
      │
      ├─ has many ─── Photo[]        (filename, url, isPrimary)  ←── stored in S3
      │                    └── used by Videos
      │
      ├─ has many ─── VoiceSample[] (filename, url, duration, isDefault)  ←── stored in S3
      │                    └── used by Videos
      │
      ├─ has many ─── Video[]       (title, script, status, model, videoUrl, thumbnailUrl)
      │                    ├── has one Schedule
      │                    └── has many AnalyticsEvents
      │
      ├─ has many ─── Schedule[]    (platform, scheduledAt, publishedAt, status)
      │
      ├─ has many ─── SocialAccount[] (platform, handle, accessToken, refreshToken)
      │                    └── unique per [userId, platform]
      │
      └─ has many ─── AnalyticsEvent[] (eventType, platform, count, date)
